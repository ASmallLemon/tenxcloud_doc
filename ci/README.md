# 持续集成
![ci](../images/ci/ci.jpg)
持续集成是一种软件开发实践，在软件开发过程中每天都会发生很多次集成。如果每次集成都能通过自动化的构建（包括编译、打包、自动化测试、发布）来验证，从而通过自动化系统尽快地发现集成错误，会大大减轻开发运维团队的工作，提高软件开发的效率。
